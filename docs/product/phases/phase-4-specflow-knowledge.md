# Phase 4：`.specflow` 知识进化

目标是让 `.specflow` 成为真正的仓库级长期知识层。

`.specflow` 记录仓库当前应该被系统和 AI 理解的事实与规则，包括项目结构、架构事实、编码规范、术语、约束、workflow 规则、历史经验总结和常见修复模式。

`.specflow` 不是 ticket 文件夹，也不是普通文档归档目录。

## 范围

- workflow run 可以提出 `.specflow` 更新建议
- 重复出现的问题可以沉淀为规则
- 历史 repair 模式可以变成未来提示
- 用户可以 review 后接受或拒绝 `.specflow` 更新
- `.specflow` 保持简短、明确、可被 agent 稳定读取

## 更新原则

agent 可以提出更新建议，但不能无提示地自动修改核心规则。

## 非目标

- 隐式黑盒记忆
- 不可审查的规则修改
- 替代人类架构判断
