# Phase 6：团队协作层

目标是让 Specflow 从本地开发工具扩展为团队级工作流产品。

## 范围

- 多用户 workflow 可见性
- 团队级 `.specflow` 治理
- 共享 workflow run 历史
- 人类审批节点
- 角色和权限
- 团队约定
- ticket、plan、review、repair 的协作
- 可选 hosted 模式

## 团队概念

- 团队 workflow 模板
- 项目级规范
- 人类 approval gate
- 共享运行历史
- 可审查的 spec evolution
- 跨 ticket 学习

## 非目标

- 替代 issue tracker
- 替代 CI/CD
- 替代产品负责人
